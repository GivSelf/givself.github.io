GivSelf

GivEnergy are introducing paid subscriptions for their cloud portal. Take back control.

Free, open-source alternative that runs on your own hardware. Monitor, control, and analyse your home energy system without monthly fees — communicating directly with your inverter over the local network.

Get Started GitHub

Why GivSelf?

💰

No Monthly Fees

GivEnergy's cloud portal is moving to a paid subscription model. GivSelf is completely free and open-source — run it on any hardware you already own.

🏠

Your Data, Your Server

All data stays on your local network. No cloud dependency for monitoring or controlling your inverter. Works even if the internet goes down.

🔓

No Lock-In

Open-source with a hardware adapter pattern. Built for GivEnergy today, extensible to other inverter brands tomorrow. You own your energy data forever.

Features

⚡

Live Monitoring

Real-time power flow with animated SVG diagram. See solar generation, battery charge/discharge, grid import/export, and house consumption updated every 10 seconds via WebSocket.

📊

Energy Analytics

Half-hourly, daily, and monthly energy flow charts from GivEnergy Cloud API. Pie chart breakdown by perspective (Home, Solar, Battery). Date navigation with full history.

🌤️

Solar Forecasting

Dual forecast sources — Forecast.Solar (free, every 30 min) and Solcast (10 calls/day). Yellow dashed overlay on analytics charts shows predicted vs actual generation.

🔋

Battery Control

ECO, Timed Demand, and Timed Export modes. 10 charge and 10 discharge time slots. Force charge/export with countdown. Reserve SOC and target SOC sliders.

📱

Mobile App

Native iOS and Android app built with Expo (React Native). Same features as the web dashboard with system theme support (light/dark/auto).

🐳

Docker Deployment

Single docker-compose file. TimescaleDB for time-series data. CI/CD via GitHub Actions publishes images to GHCR on every push to main.

🔌

Local Modbus TCP

Direct communication with your inverter over the local network. No cloud dependency for live data. Ported from the GivTCP protocol implementation.

📥

Historical Import

Import years of historical energy flow data from the GivEnergy Cloud API. UI-driven with live progress bar. Configurable date range.

⚙️

Setup Wizard

First-run quickstart walks through inverter connection, cloud API, and solar forecasting setup. All settings persisted to database — no .env files needed.

Quick Start

# Create a docker-compose.yml with this content, then:
docker compose pull
docker compose up -d
# Open http://your-server:3033 and follow the setup wizard

See the deploy repo for the full docker-compose.yml template.

Docker Compose

# docker-compose.yml
services:
  timescaledb:
    image: timescale/timescaledb:latest-pg16
    environment:
      POSTGRES_DB: givself
      POSTGRES_USER: givself
      POSTGRES_PASSWORD: your_password
    volumes:
      - db_data:/home/postgres/pgdata/data

  server:
    image: ghcr.io/givself/server:latest
    depends_on: [timescaledb]
    environment:
      PORT: "3032"
      DATABASE_URL: postgres://givself:your_password@timescaledb:5432/givself
      ADAPTER_TYPE: givenergy
      INVERTER_HOST: your_inverter_ip

  web:
    image: ghcr.io/givself/web:latest
    ports: ["3033:3000"]
    environment:
      HOSTNAME: "0.0.0.0"
      API_URL: "http://server:3032"
      WS_URL: "ws://your_server_ip:3032"

volumes:
  db_data:

Architecture

Microservice architecture with independent repos, Protobuf contracts, and CI/CD.

📋 contracts

Protobuf definitions + generated TypeScript types. Buf toolchain with lint + breaking change detection.

🖥️ server

Fastify backend. Modbus TCP adapter, GivEnergy Cloud API, Solcast + Forecast.Solar, TimescaleDB.

🌐 web

Next.js 16 dashboard. Recharts analytics, animated power flow, dark/light theme, mobile responsive.

📱 mobile

Expo (React Native) app for iOS and Android. Victory Native charts, system theme support.

🐳 deploy

Docker Compose orchestration. TimescaleDB, server, web. CI builds + publishes to GHCR.
